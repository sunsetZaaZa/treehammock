# Raptor Balcony 1.0.0 Release Feature Matrix

This document is the 1.0.0 feature source of truth. The README gives the short operating summary, this matrix owns feature status, and `docs/sql/SQL_CONTRACT_MATRIX_1_0_0.md` owns the PostgreSQL function contract.

## Status legend

| Status | Meaning |
|---|---|
| Complete | Required for 1.0.0 and implemented with source or contract tests. |
| Limited | Implemented for a deliberately narrow 1.0.0 scope. Anything outside the listed boundary is unsupported. |
| Future | Not part of 1.0.0. Do not wire config, endpoints, or providers for it without a new design PR. |
| Unsupported | Intentionally rejected and should fail closed. |

## ABUSE-10 required authenticated mutation idempotency

Selected authenticated mutating endpoints require an `Idempotency-Key` header. The API scopes the key to account id, HTTP method, and route, stores only a fingerprinted key in Dragonfly, reserves the operation while the first request is running, and stores short-lived reservation/completion markers for 10 to 15 minutes. Duplicate requests with the same key do not repeat side effects. `POST /account/logoff` remains optional because logout should be naturally repeatable.

Required endpoints:

- `POST /account/setuptwofactormethod`
- `POST /account/adjust`
- `POST /account/wipeout`
- `POST /account/wipeout/finalize`
- `POST /account/logoff/all`
- `POST /account/sessions/revoke`
- `POST /activations/place`
- `POST /activations/disable`

Optional endpoint:

- `POST /account/logoff`

The raw `Idempotency-Key` value is forbidden in logs, metrics, and cache keys.

## SYSTEM-FLOW-1B system coverage

The Docker/host-backed system suite now exercises setup of email, SMS, and authenticator-app 2FA plus login using each verified method through HAProxy. Email and SMS are captured by SystemTesting-only providers; authenticator-app verification uses real RFC 6238 TOTP generated from the setup manual-entry key.
